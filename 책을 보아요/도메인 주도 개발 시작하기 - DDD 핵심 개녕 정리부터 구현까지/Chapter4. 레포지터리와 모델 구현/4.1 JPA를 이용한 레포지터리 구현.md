# JPA 를 이용한 레포지터리 구현
만약 데이터 저장소를 RDBMS 를 사용한다면, 객체 기반의 도메인 모델과 관계형 데이터 모델을 매핑해주는 기술로 ORM 이 있다.
이번 챕터에서는 자바의 ORM 표준인 JPA 를 사용하여 레포지터리와 애그리거트를 구현해보자.


## @OneToMany
- 애그리거트에서 엔티티에 대해 매핑할때..
- 예시. 주문과 주문 상품은 1:N 관계로 부모는 자식을 알아야 하므로 주문은 주문 상품을 가진다.
이 때, JPA 에서 매핑할 떄는.. 양방향으로 하거나, 부모에서 단방향으로 한다.

1. 양방향
2. 단방향
- 부모에서 자식 필드에 대해 @OneToMany 를 달고, mappedBy 를 쓰지 않는다면 @JoinColumn 도 같이 작성한다
joinColumns 을 쓰지 않는다면 단방향 OneToMany는 외래키(FK)를 자식 테이블에 두지 못하여 자동으로 정션 테이블을 생성하게 된다. 즉 조인 컬럼은 해당 컬럼을 자식 테이블에 필드를 생성하는 것.

mappedBy

단방향에선 사용하지 않는다.

mappedBy 는 “양방향 관계에서 FK를 관리하는 쪽을 지정하는 속성”
