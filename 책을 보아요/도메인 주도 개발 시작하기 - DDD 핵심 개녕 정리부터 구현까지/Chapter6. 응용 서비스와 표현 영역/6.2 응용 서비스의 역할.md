# 6.2 응용 서비스의 역할
응용 서비스는 사용자가 요청한 기능을 수행하기 위해 도메인 객체를 리포지터리에서 가져오고 도메인 객체 간의 흐름을 제어한다.

```java
public Result createProduct(CreateProductReq req) {
    // 1. 데이터 중복 등 유효성 검사
    validate(req)

    // 2. 애그리거트 생성
    Product newProduct = create(req);

    // 3. 애그리거트 저장
    productRepository.save(newProduct);

    // 4. 결과 리턴
    return createSuccessfulResult(newProduct);
}
```
주로 흐름 제어를 하기에 단순한 형태를 갖는다. </br>
응용 서비스가 복잡하다면 응용 서비스에서 도메인 로직의 일부를 구현했을 가능성이 있다. </br>


응용 서비스는 트랜잭션 처리를 담당하여 데이터의 일관성을 지킬 수 있다. </br>
이 외에도 접근 제어, 이벤트 처리를 수행하는데 뒤에서 살펴본다.

### 응용 서비스에 도메인 로직 넣지 않기
첫번째, 코드의 응집성이 떨어진다.
도메인 데이터와 그 데이터를 조작하는 로직이 한 영역에 위치하지 않는다면, 로메인 로직을 파악하기 위해 여러 영역을 확인해야한다.

두번째, 코드 중복이 늘어난다.
여러 응용 서비스에 동일한 도메인 로직을 구현할 가능성이 높기 때문이다.

도메인 로직은 도메인 영역에 모으도록 하자.
