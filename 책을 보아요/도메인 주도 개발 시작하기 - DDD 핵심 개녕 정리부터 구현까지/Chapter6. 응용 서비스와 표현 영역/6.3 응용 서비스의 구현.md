# 응용 서비스의 구현
응용 서비스 구현 시 고려 사항과 트랜잭션과 같은 구현 기술의 연동에 대해 알아본다.

## 응용 서비스의 크기
응용 서비스는 보통 아래의 두 가지 방법 중 하나로 구현할 수 있다.
1. `하나의 응용 서비스에 도메인의 모든 기능 구현하기`
2. `기능 별로 응용 서비스 클래스를 따로 구현하기`

1번의 장점/단점이 2번의 단점/장점이 된다.

1번의 장점과 단점을 작성해보고, 각 요소에 대한 2번의 특징을 작성하겠다.
- (장점) 기능 별 동일한 로직을 하나의 클래스에서 구현하여 중복 로직을 제거할 수 있다.
    - 2번 방식에서는 별도 클래스에 공통 로직을 작성하여 중복 코드를 제거할 수 있다.
- (단점) 하나의 클래스의 크기가 커지고 연관성이 없는 코드, 필드가 늘어난다.
    - 2번 방식에서는 클래스의 개수는 많아지지만, 코드 품질을 일정 수준으로 유지하는데 도움이 된다.

```java
// 1 번 방식
public class UserService {
    public void leave() {...};
    public void join() {...};
    public void changePassword() {...};
}

// 2 번 방식
public class ChangePasswordService {
    public void changePassword() {...};
}
```
